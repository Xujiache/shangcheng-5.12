## 1. 原始需求（用户原话）

> 做一个**原生微信小程序的新一端 —— 记账**。需要支持**在管理后台增加账号、增加会员时长**。原型图已画好（`门窗利账.html`）。

补充自设计 chat 记录的最终落点（用户迭代 8 轮后的定稿）：

1. 登录从「微信一键登录」改为**手机号 + 密码（以密码为主）**，验证码登录为辅，微信登录作「其他方式」保留。
2. **会员制**：只有管理后台能设置某账号是不是会员、以及多久的会员（1天/7天/30天/90天/365天均可单独设置）；**过期即不可用**；要有独立会员页展示期限、个人中心展示、登录时提示；**不破坏原有 UI 颜值**。

## 2. 项目上下文

- 现有 monorepo `jiujiu-mall`（pnpm，`packages/*`）：4 前端（user-mp/merchant-app/platform-app 为 uni-app；admin-pc 为 Vue3+Vite）+ NestJS 后端 + shared 共享层。
- 后端：NestJS 10 + Prisma 5 + PostgreSQL + Redis + MinIO；统一响应壳 `{code,data,message,msg,traceId,timestamp}`；全局 `JwtAuthGuard` + `RolesGuard`；前缀 `/api/v1`。
- admin-pc：商家/平台/超管三合一智能登录；平台工作台已有「会员套餐」等管理页可参考。
- 这是一个**与商城业务无关的全新产品**（门窗店主的订单利润记账），只在「账号 + 会员」和「admin-pc 后台 + 后端基础设施」层面与现有系统共用。

## 3. 需求理解（这是什么 App）

门窗加工/安装店主用的**订单利润记账工具**：录入每单收款总价 + 各项成本，自动算利润，按月/季/年看经营报表。

- 成本 = 5 项固定类（型材/玻璃/配件/人工/纱窗）+ 其他开销（运费/上门安装费/油费/测量费/其他）。
- 利润 = 总价 − 总成本；利润率 = 利润 / 总价。
- 客户为独立档案（姓名/电话/地址/备注），下单时选择或新增。
- 会员闸门：登录后若未开通/已过期 → 进「开通会员」页且功能锁定；剩 ≤7 天弹提示。

设计已含 **28 个页面/路由**（见 DESIGN 屏幕清单），全链路闭环。

## 5. 边界确认

**在范围内**

- 新端小程序（`packages/ledger-mp`）：登录 + 会员闸门 + 记账核心 + 报表 + 客户 + 设置全链路（28 屏）。
- 后端 `ledger` 域：App 接口（`/api/v1/l/*`）+ 后台管理接口（`/api/v1/p/ledger/*`）。
- admin-pc：新增「门窗利账」管理（账号增删/改密/禁用、会员时长增减与查看）。

**不在范围内（本期）**

- App 内微信支付续费（D3 已定无支付）。
- 微信一键登录的真实 openid 绑定（先做手机号+密码；微信登录入口保留为占位/后续）。
- 小程序正式上架（AppID 申请、审核材料）—— 列为收尾 TODO。
- 商城侧任何改动（完全隔离，零侵入）。

## 6. 假设（如不符请纠正）

- A1：每个记账账号的订单/客户/目标数据**互相隔离**（私人记账），后端所有查询按 JWT 中的 `ledgerUserId` 强制 scope。
- A2：后台管理入口归**超管/平台角色**（你本人 `super@demo`）；记账账号由后台创建，**App 端无自助注册**。
- A3：技术品牌沿用 `@jiujiu/*` 命名惯例；新包 `@jiujiu/ledger-mp`、后端域 `ledger`、路由前缀 `/l`。
- A4：会员「今天」以**服务端时间**为准计算到期与剩余天数（设计里硬编码的 2026-06-08 仅原型用）。
